TOUR-04 — stage-level GameMatchType override. Two surfaces:

1. BracketMatchMaterialiserService: when spawning a bracket's GameMatch, prefer
   stage.game_match_type_id over tournament.default_game_match_type_id. Keep the
   existing "no match type anywhere" RuntimeException as the final guard.

   set the override per stage, plus a read-only column showing the effective type.
   The Select offers only the tournament's game's match types (Pattern 3 cross-game
   guard, mirrors RoleLimitsRelationManager). The existing read-only invariant on
   ordinal/type/name is preserved — only game_match_type_id becomes editable.

files with them, so it runs in parallel (Wave 3).

Purpose: TOUR-04's SC is "admin sets a different type on a stage; matches in that
stage use it". Output: 1 modified service, 1 modified Filament RM, 1 lang edit, 2 GREEN tests.
