<objective>
Replace the bot's two redirect-to-web stubs with live API calls (CLAN-02), and teach `translateError` the three new error codes (CLAN-03 localized Discord reasons).

The `/clan apply <slug>` slash command and the `clan_apply` button both currently editReply a "use the website" stub. This plan flips them to `api.post('/clans/<id>/applications', {}, { actsAsDiscordId })`, surfacing a success confirmation or a translated error. The bot stays a thin display layer (D-004) — zero business logic, all eligibility decided server-side by the service (plan 10-02) and surfaced as 422 error codes the bot maps to user copy.

Purpose: make Discord a first-class application surface.
Output: 2 bot source edits, 2 bot test edits.

This plan touches ONLY `apps/bot/*` — it runs in parallel with the web service/i18n plan (10-02). The bot tests mock `api.post`, so they do not require the web endpoint (plan 10-03) to be deployed.
</objective>

|----------|-------------|
| Discord user → bot → /api/bot (acts-as) | The bot forwards interaction.user.id as actsAsDiscordId; the web side rebinds + decides eligibility |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-05-01 | Spoofing | bot acting as the wrong user | mitigate | actsAsDiscordId = interaction.user.id (Discord-verified invoker); the web bot.acts-as middleware rebinds auth — the bot cannot assert an arbitrary applicant |
| T-10-05-02 | Information disclosure | API error bodies surfaced to Discord | mitigate | translateError maps known codes to fixed copy; unknown errors fall through to `Failed: <scrubbed>` — api.ts already token-scrubs the message (T-05-08-01) before it reaches here |
| T-10-05-03 | Tampering | bot embedding business logic | mitigate | The bot only posts + translates; all guards live in ClanApplicationService (D-004 thin display layer) |
| T-10-05-SC | Tampering | npm installs | accept | No package installs in this plan — pure TS edits against existing deps |
