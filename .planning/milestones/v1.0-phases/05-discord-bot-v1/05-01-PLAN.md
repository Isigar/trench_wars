|----------|-------------|

| docker-compose worker -> redis | Horizon worker is a privileged process (can fail any job) — runs as the same user as web (D-021) |
| .env.example -> production | WEB_API_TOKEN is empty in .env.example; never commit a token-shaped placeholder (Pitfall 3) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | Tampering | RED stub still passing trivially — false GREEN | mitigate | Stubs use markTestIncomplete which counts as incomplete not pass; baseline RED count documented in SUMMARY |
| T-05-01-02 | Information Disclosure | committed token-shaped placeholder in .env.example | mitigate | .env.example line MUST be `WEB_API_TOKEN=` (empty) — Pitfall 3 explicit guidance |
| T-05-01-03 | DoS | Horizon worker container in tight crash loop | mitigate | healthcheck pgrep; restart policy default (Phase 1 idiom — no explicit restart added) |
| T-05-01-04 | Spoofing | Sanctum personal_access_tokens.token stored plaintext | mitigate | Sanctum default hashes tokens at rest (SHA-256); plan 05-07 issuance command MUST log the plaintext ONCE to operator, never persist anywhere |
| T-05-01-05 | Tampering | install:api appends api: route group middleware list including stateful Sanctum | mitigate | Pitfall 4 verification — task includes config:show sanctum.stateful check; ensure web-nginx + bot containers are NOT in the stateful list |
