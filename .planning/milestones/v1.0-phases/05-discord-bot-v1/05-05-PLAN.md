<objective>
Wave 4: extend the Phase 4 MatchObserver (which already syncs the polymorphic Event row) so it ALSO writes a pending discord_outbound_messages row whenever a public match is created/updated AND the host clan has a configured discord_announce_channel_id. This is SC-3's web-side trigger — without this hook, the bot's outbound poller (plan 05-11) would never see anything to dispatch.

The observer creates ONE outbound row per relevant event:
- created(): a `match_announce` row that the bot delivers as an embed + RSVP buttons (plan 05-11 renders + sends; plan 05-10 builds embed/button helpers)
- updated() when status transitions: a `match_announce` row with `sent_message_id` from the prior announce, so plan 05-11's worker can EDIT the original message (idempotent UX — no spamming the channel with re-posts)

DiscordOutboundPayloadBuilder is a small support class that owns the canonical JSONB payload shape. Keeping the shape in one place means plan 05-11 (bot-side renderer) reads a stable contract from `@trenchwars/shared-types` (the payload doesn't need DTO regeneration for now — it's a free-form JSONB; plan 05-12 promotes it to a DTO if a type contract is needed).

The plan does NOT touch the pre-existing Phase 4 Event sync side-effect — that flow stays as-is (PublicMatchData -> events table). The outbound write is an ADDITIVE side-effect inside the same observer hooks.

Purpose: SC-3 — "when a match is created on the website, the host clan's announce channel receives an embed with RSVP buttons, persisted in discord_outbound_messages".

Output: extended MatchObserver + new payload builder + 1 GREEN test file.
</objective>

Extended MatchObserver::created body (pseudocode preserving Phase 4 logic + adding outbound):

  public function created(GameMatch $match): void {
      // Phase 4 logic — preserve (Event sync via PublicMatchData)
      ...existing Phase 4 Event sync code...

      // Phase 5 addition — write match_announce outbound row if eligible
      $this->writeMatchAnnounceIfEligible($match, isUpdate: false, priorSentMessageId: null);
  }

  public function updated(GameMatch $match): void {
      ...existing Phase 4 Event sync code...

      // Phase 5 — only fire outbound update when status transitions (cheap re-eval)
      if ($match->wasChanged('status')) {
          $prior = DiscordOutboundMessage::query()
              ->where('message_type', 'match_announce')
              ->where('status', 'sent')
              ->where('payload->match_id', $match->id)
              ->orderByDesc('updated_at')
              ->first();
          $this->writeMatchAnnounceIfEligible($match, isUpdate: true, priorSentMessageId: $prior?->sent_message_id);
      }
  }

  private function writeMatchAnnounceIfEligible(GameMatch $match, bool $isUpdate, ?string $priorSentMessageId): void {
      if (! $match->is_public) return;
      $channelId = $match->hostClan?->discord_announce_channel_id;
      if ($channelId === null) return;

      $payload = $isUpdate
          ? DiscordOutboundPayloadBuilder::buildMatchUpdate($match, $priorSentMessageId)
          : DiscordOutboundPayloadBuilder::buildMatchAnnounce($match);

      DiscordOutboundMessage::create([
          'channel_id' => $channelId,
          'message_type' => 'match_announce',
          'status' => 'pending',
          'payload' => $payload,
          'causer_user_id' => auth()->id(),
      ]);
  }

DiscordOutboundPayloadBuilder — stateless static helper:

  namespace App\Support
  final class DiscordOutboundPayloadBuilder

  public static function buildMatchAnnounce(GameMatch $match): array {
      $match->loadMissing(['gameMatchType','hostClan','slots.role']);
      return [
          'kind' => 'match_announce_new',
          'match_id' => $match->id,
          'status' => $match->status,
          'is_public' => $match->is_public,
          'scheduled_at' => $match->scheduled_at?->toIso8601String(),
          'host_clan_id' => $match->host_clan_id,
          'host_clan_name' => $match->hostClan?->name,
          'game_match_type_id' => $match->game_match_type_id,
          'game_match_type_key' => $match->gameMatchType?->key,
          'title' => $match->getTranslation('title', 'en'),
          'slot_summary' => self::buildSlotSummary($match),
      ];
  }

  public static function buildMatchUpdate(GameMatch $match, ?string $priorSentMessageId): array {
      $base = self::buildMatchAnnounce($match);
      $base['kind'] = 'match_announce_update';
      $base['prior_sent_message_id'] = $priorSentMessageId; // bot edits existing message if non-null
      return $base;
  }

  private static function buildSlotSummary(GameMatch $match): array {
      // Group slots by game_role_id: {role_id, role_key, role_display, total, filled}
      return $match->slots->groupBy('game_role_id')->map(function ($group) {
          $first = $group->first();
          return [
              'role_id' => $first->game_role_id,
              'role_key' => $first->role?->key,
              'role_display' => $first->role?->getTranslation('display_name', 'en'),
              'total' => $group->count(),
              'filled' => $group->whereNotNull('occupant_user_id')->count(),
          ];
      })->values()->all();
  }

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MatchObserver -> DiscordOutboundMessage | Observer is the ONLY production write path for match_announce rows; bot has NO direct access (D-004) |
| Match.is_public + hostClan.discord_announce_channel_id -> outbound channel | Operator-controlled (Filament edits clan profile); never user-controllable |
| auth()->id() captured -> outbound.causer_user_id | Request-scope attribution; CLI seeders write null causer (acceptable) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05-01 | Tampering | Private match leaks via outbound row | mitigate | is_public guard at the top of writeMatchAnnounceIfEligible — private match never produces an outbound row |
| T-05-05-02 | Information Disclosure | Match title with sensitive content broadcast to public channel | accept | Match title is set by admin/officer with audit-log accountability; the public channel is operator-supplied per clan; out-of-band moderation |
| T-05-05-03 | Spoofing | causer_user_id absent — outbound row attributed to system | accept | LogsActivity on DiscordOutboundMessage captures the event; CLI seeders are infrequent and explicit |
| T-05-05-04 | DoS | Mass UPDATE on matches triggers thousands of outbound rows | mitigate | Update-side fires only when status changes (wasChanged check); title-only edits do NOT fire outbound |
| T-05-05-05 | Tampering | bot worker dispatches an old prior_sent_message_id and edits unrelated channel content | mitigate | sent_message_id is scoped to the bot's own past output; replay-safe by design (Discord rejects edit of non-existent message gracefully) |
| T-05-05-06 | Tampering | Observer fires twice for the same match -> duplicate announce | mitigate | Phase 4 D-04-08-B model-level booted() registration ensures observer fires ONCE per save |
