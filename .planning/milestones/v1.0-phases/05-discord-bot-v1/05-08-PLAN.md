This is the substrate every later bot-side plan depends on. Slash commands (plan 05-09) call `api.get('/matches/...', { actsAsDiscordId })`; component handlers (plan 05-10) call `decodeButtonId(interaction.customId)` to route; the outbound worker (plan 05-11) calls `api.get('/outbound-messages?status=pending')` + `api.post('/outbound-messages/:id/sent', ...)`.

The api client is the critical piece. It MUST scrub the token from any error trace (Pitfall 3 — `WEB_API_TOKEN` is plaintext in env; if logged in a stack trace it's effectively leaked). The implementation does a `text.replace(env.WEB_API_TOKEN, '[REDACTED]')` before throwing.

The Client intent configuration is the second critical piece. `GuildMembers` is a PRIVILEGED intent — must be enabled in the Discord Developer Portal AND requested at boot. Without it, `guildMemberUpdate` events never fire (Pitfall 6) and SC-4 reconciliation silently fails. Documentation lands in the operator smoke list (VALIDATION.md) — code-side we just declare the intent.

Purpose: every Phase 5 plan after this one consumes one or more of these substrates. Without this plan, the bot side has no `api.get()` to call.

  // Reads from process.env at boot; throws if required values missing.
  function required(key: string): string {
      const v = process.env[key];
      if (v === undefined || v === '') {
          throw new Error(`[bot/env] Missing required env var: ${key}`);
      }
      return v;
  }

  function optional(key: string, fallback: string): string {
      return process.env[key] ?? fallback;

apps/bot/src/services/api.ts (RESEARCH Example 1 verbatim with strict TypeScript):

  import { fetch, Headers } from 'undici';
  import { env } from '../env';

  interface CallOptions {
      actsAsDiscordId?: string;
      body?: unknown;
      method?: 'GET' | 'POST' | 'DELETE';
  }

  export const api = {
      async request<T>(path: string, opts: CallOptions = {}): Promise<T> {
          const headers = new Headers({
              Accept: 'application/json',
              Authorization: `Bearer ${env.WEB_API_TOKEN}`,
          });
          if (opts.body !== undefined) headers.set('Content-Type', 'application/json');
          if (opts.actsAsDiscordId !== undefined) {
              headers.set('X-Bot-Acts-As-User', opts.actsAsDiscordId);
          }

          const res = await fetch(`${env.WEB_API_URL}/api/bot${path}`, {
              method: opts.method ?? (opts.body !== undefined ? 'POST' : 'GET'),
              headers,
              body: opts.body !== undefined ? JSON.stringify(opts.body) : undefined,
          });

          if (!res.ok) {
              const text = await res.text().catch(() => '<unreadable>');
              const scrubbed = text.replaceAll(env.WEB_API_TOKEN, '[REDACTED]');
              throw new Error(
                  `Bot API ${opts.method ?? 'GET'} ${path} -> ${res.status}: ${scrubbed.slice(0, 500)}`,
              );
          }
          return (await res.json()) as T;
      },
      get<T>(path: string, opts: Omit<CallOptions, 'method' | 'body'> = {}): Promise<T> {
          return this.request<T>(path, { ...opts, method: 'GET' });
      },
      post<T>(path: string, body: unknown, opts: Omit<CallOptions, 'method' | 'body'> = {}): Promise<T> {
          return this.request<T>(path, { ...opts, method: 'POST', body });
      },
      delete<T>(path: string, opts: Omit<CallOptions, 'method' | 'body'> = {}): Promise<T> {
          return this.request<T>(path, { ...opts, method: 'DELETE' });
      },
  };

apps/bot/src/lib/customIds.ts (RESEARCH Example 2 verbatim — Pitfall 5 mitigation):

  export type ButtonAction =
      | { kind: 'match_signup'; matchId: string; gameRoleId: string }
      | { kind: 'match_leave'; matchId: string; gameRoleId: string }
      | { kind: 'match_open_signup_modal'; matchId: string }
      | { kind: 'clan_apply'; clanId: string };

  export function encodeButtonId(a: ButtonAction): string {
      switch (a.kind) {
          case 'match_signup': return `m:s:${a.matchId}:${a.gameRoleId}`;
          case 'match_leave': return `m:l:${a.matchId}:${a.gameRoleId}`;
          case 'match_open_signup_modal': return `m:o:${a.matchId}`;
          case 'clan_apply': return `c:a:${a.clanId}`;
      }
  }

  export function decodeButtonId(s: string): ButtonAction | null {
      const parts = s.split(':');
      if (parts[0] === 'm' && parts[1] === 's' && parts.length === 4)
          return { kind: 'match_signup', matchId: parts[2]!, gameRoleId: parts[3]! };
      if (parts[0] === 'm' && parts[1] === 'l' && parts.length === 4)
          return { kind: 'match_leave', matchId: parts[2]!, gameRoleId: parts[3]! };
      if (parts[0] === 'm' && parts[1] === 'o' && parts.length === 3)
          return { kind: 'match_open_signup_modal', matchId: parts[2]! };
      if (parts[0] === 'c' && parts[1] === 'a' && parts.length === 3)
          return { kind: 'clan_apply', clanId: parts[2]! };
      return null;
  }

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process.env -> bot runtime | Fail-fast at boot if required env missing; no fallback to development defaults that could leak to prod |
| Bot -> Discord gateway | discord.js handles WS auth via DISCORD_BOT_TOKEN; never log token; intents are minimal-necessary |
| Bot -> Web API | Sanctum Bearer in Authorization header; X-Bot-Acts-As-User on per-call basis (only set when acting on behalf of a user) |
| Bot stderr/stdout -> Railway logs | Token scrubbing on every error throw (Pitfall 3); console.error of unscrubbed token would leak to log aggregator |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-08-01 | Information Disclosure | api.ts throws Error with token visible in stack trace | mitigate | text.replaceAll(env.WEB_API_TOKEN, '[REDACTED]') before throw (Pitfall 3 verbatim) |
| T-05-08-02 | Tampering | Bot starts with empty WEB_API_TOKEN and silently fails all API calls | mitigate | env.ts required() throws at boot if WEB_API_TOKEN is empty |
| T-05-08-03 | EoP | customId forged to point at arbitrary matchId / gameRoleId | accept | The downstream signup endpoint (plan 05-04) enforces D-010 row-locked capacity + tag access; forging a customId yields a 422 error, not a successful overflow |
| T-05-08-04 | DoS | Malformed customId crashes decoder | mitigate | decodeButtonId returns null on malformed input; downstream router treats null as "unknown button — ignore" |
| T-05-08-05 | Spoofing | Bot logs in with wrong DISCORD_APPLICATION_ID and registers commands to wrong app | mitigate | env.ts required() throws if missing; plan 05-09 registerCommands uses applicationGuildCommands(applicationId, guildId) — wrong applicationId = Discord 401 |
| T-05-08-06 | Tampering | Privileged intent missing — guildMemberUpdate never fires | mitigate | client.ts requests GuildMembers intent; operator manual smoke confirms Server Members Intent ON in Dev Portal (VALIDATION.md) |
| T-05-08-07 | Replay | uncaughtException + process.exit(1) terminates bot leaving outbound rows in dispatching state | accept | Railway/docker restart policy brings bot back; on restart, the lockForUpdate semantics + the dispatching-state-too-long admin cleanup (Phase 9 polish) handle stale rows |
