## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Migration runtime to Postgres | Raw `DB::statement` executes unparameterised SQL — content must be literal strings, not user input |
| App code to clan_memberships writes | Partial unique index is the ONLY reliable enforcement at concurrent write time (D-009) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Tampering | clan_memberships partial unique index | mitigate | Raw `CREATE UNIQUE INDEX ... WHERE left_at IS NULL` — DB rejects duplicate active rows; Pest test in plan 02-03 asserts QueryException |
| T-02-01-02 | Information Disclosure | clans + clan_memberships FKs | mitigate | `restrictOnDelete` on FKs to `users` and `clans` — prevents cascading deletion of audit trail; soft deletes on clans only |
| T-02-01-03 | Tampering | clan status / membership role enum | mitigate | CHECK constraints in raw SQL — DB rejects illegal status / role values regardless of $fillable bypass |
| T-02-01-04 | Repudiation | timestamptz columns | mitigate | created_at/updated_at upgraded to timestamptz with UTC interpretation — prevents drift across regions |
| T-02-01-05 | Elevation of Privilege | discord_guild table | mitigate | Singular table name + no public route + admin-only Filament resource in plan 02-13 (no-Create getPages()) — operational gate, not pure DB |
</threat_model>
