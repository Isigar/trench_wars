<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Inertia props to Vue render | Vue templates HTML-escape by default; v-html is forbidden on description / bio |
| Filter bar query params (Inertia router.get) | Strings — passed through; backend re-validates |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08-01 | Tampering | XSS via clan description | mitigate | Description is rendered via `{{ description.en }}` (Vue auto-escapes); no `v-html` |
| T-02-08-02 | Information Disclosure | Privacy logic in Vue | mitigate | Pages use `v-if="prop !== undefined"`, never `v-if="player.showFlag"`. UI-SPEC Pitfall 2. |
| T-02-08-03 | Tampering | Inertia router.get with attacker-controlled query | mitigate | Backend (plan 02-07) validates `tag` and `q` params; Vue just URL-encodes |
| T-02-08-04 | Information Disclosure | Hardcoded EN strings | mitigate | NoHardcodedStringsTest covers `resources/js/pages/Clans/`, `Players/`, `MyClan/` |
</threat_model>
