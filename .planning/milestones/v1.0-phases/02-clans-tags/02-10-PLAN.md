<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Leader/Officer to invite target | Must verify target user exists and has no active clan_memberships |
| Invitee to invite | Must verify acting user is invited_user_id of the pending invite |
| Invite accept to membership creation | Must be atomic — partial state (invite=accepted, membership not created) is fatal |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06-01 | Spoofing | Accept invite for wrong user | mitigate | Accept controller asserts `$invite->invited_user_id === auth()->id()` else abort 403 |
| T-02-06-02 | Tampering | Re-accept already-accepted invite | mitigate | Service checks `$invite->status === 'pending'`; rejects other states |
| T-02-06-03 | Tampering | Concurrent dual-invite acceptance (race) | mitigate | DB::transaction wraps invite update + membership insert; D-009 partial unique index on clan_memberships rejects the second create |
| T-02-06-04 | DOS | Mass invite spam | accept | P2 has no rate limit; Phase 9 may add per-clan-per-day caps |
| T-02-06-05 | Information Disclosure | Listing all invites globally | mitigate | Index of /my-clan invites is scoped to actor's clan via Policy |
| T-02-06-06 | Repudiation | Causer null on system expiry | accept | Expiry is not wired in P2; future cron job will set causer=null which is correct for system-driven transitions |
</threat_model>

    Algorithm for accept():
    ```
    if ($acceptor->id !== $invite->invited_user_id) abort(403);
    if ($invite->status !== 'pending') throw new DomainException(__('clans.invites.error.not_pending'));
    if (ClanMembership::where('user_id', $acceptor->id)->whereNull('left_at')->exists()) {
        throw new DomainException(__('clans.invites.error.already_in_clan'));
    }
    DB::transaction(function () use ($invite, $acceptor): ClanMembership {
        $invite->update(['status' => 'accepted', 'decided_at' => now()]);
        return ClanMembership::create([
            'clan_id'   => $invite->clan_id,
            'user_id'   => $acceptor->id,
            'role'      => 'recruit', // new joiners start as Recruit; Leader/Officer promote later
            'joined_at' => now(),
            'left_at'   => null,
            'invited_by'=> $invite->inviting_user_id,
        ]);
    });
    ```
