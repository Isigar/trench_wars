    Build `DiscordGuildSeeder` mirroring the PermissionSeeder.php structure (declare, namespace, class extends `Illuminate\Database\Seeder`, single `run()` method, no constructor). Inside `run()`:
    ```
    DiscordGuild::firstOrCreate([], [
        'guild_id'  => null,
        'name'      => null,
        'icon_url'  => null,
    ]);
    ```
    The empty first arg is THE singleton trick — see Eloquent docs: `firstOrCreate([])` returns the first row when one exists, else inserts. With the table starting empty, the first invocation inserts; subsequent invocations find the existing row and return it. The DiscordGuild migration has no NOT NULL columns other than `id` (defaulted) and timestamps (auto), so the insert succeeds with all nulls. Admin fills `guild_id` via Filament edit later (plan 02-13).
