Canonical Phase 4 MatchStatusService idiom (verified `apps/web/app/Services/MatchStatusService.php`):

```php
<?php
declare(strict_types=1);
namespace App\Services;

use App\Models\{Model};
use App\Models\User;
use Illuminate\Support\Facades\DB;

final class {Model}StatusService
{
    /** @var array<string, list<string>> */
    private const ALLOWED = [...];

    public function transition({Model} $m, string $to, ?User $causer = null): {Model}
    {
        $from = $m->status;
        if (!in_array($to, self::ALLOWED[$from] ?? [], true)) {
            throw new {Model}StatusInvalidTransitionException(__('{namespace}.errors.invalid_transition', ['from' => $from, 'to' => $to]));
        }
        return DB::transaction(function () use ($m, $from, $to, $causer): {Model} {
            $m->update(['status' => $to]);
            activity()
                ->causedBy($causer ?? auth()->user())
                ->performedOn($m)
                ->withProperties(['from' => $from, 'to' => $to])
                ->log("{Model} status: {$from} -> {$to}");
            return $m;
        });
    }
}
```

private const ALLOWED = [

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Caller to transition() | Service is stateless; caller MUST pass User $causer (optional; auth()->user() fallback for HTTP routes — system-initiated transitions in observers/cron pass explicit causer) |
| Service to activity_log | Service writes audit log via causer-bound activity()->log() inside the same DB::transaction as the status update — partial state is impossible |
| Service to DB | DB::transaction wraps both writes; if activity()->log() throws, status update rolls back |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04-01 | Tampering | Invalid status transition via direct $tournament->update(['status'=>'foo']) bypass | mitigate | DB-layer tournaments_status_check CHECK (plan 06-02) rejects any string not in the 6-value enum; service-layer ALLOWED matrix rejects valid-string-but-invalid-transition (e.g., completed -> running) |
| T-06-04-02 | Repudiation | Status flip with no audit trail | mitigate | activity()->causedBy($causer)->withProperties(['from','to'])->log() inside DB::transaction — partial state impossible (Phase 4 D-04-04 precedent) |
| T-06-04-03 | Tampering | Filament admin manually edits Tournament.status via form | mitigate | Filament TournamentResource (plan 06-11) disables() the status field on Edit; transitions go through dedicated Filament Actions that route through this service (RESEARCH Pattern 10) |
| T-06-04-04 | Information Disclosure | Audit log leaks internal status strings to non-admin viewers | accept | Status values are part of the public Tournament contract (rendered on /tournaments page via status badges); not sensitive |
| T-06-04-05 | Spoofing | Caller passes a User they don't own as the causer | accept | This is a service-layer call (not HTTP-driven directly); upstream Filament action callers use $this->record + auth()->user() pattern; if a malicious code path lies about the causer, the threat is upstream of this service |
