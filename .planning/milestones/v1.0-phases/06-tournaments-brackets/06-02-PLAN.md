Canonical Phase 4 migration shape (verified Phase 4 plan 04-02 commits):

- Filename: `YYYY_MM_DD_HHMMSS_create_{table}_table.php` (Laravel default; use 2026_05_15_100000 / 100100 / 100200 / 100300 / 100400 for stable ordering)
- File body: anonymous class returning Migration; `up()` runs Schema::create() + DB::statement post-creates for timestamptz / CHECKs / DEFAULTs that Schema can't express; `down()` reverses via Schema::dropIfExists()
- ALWAYS: `$table->uuid('id')->primary();` THEN `DB::statement('ALTER TABLE {table} ALTER COLUMN id SET DEFAULT gen_random_uuid();');` after the Schema::create block
- ALWAYS: `$table->timestamps();` THEN `DB::statement("ALTER TABLE {table} ALTER COLUMN created_at TYPE timestamptz USING created_at AT TIME ZONE 'UTC';")` + same for updated_at
- CHECK constraints via `DB::statement("ALTER TABLE {table} ADD CONSTRAINT {table}_{name}_check CHECK (...)");` — Postgres identifier length limit 63 bytes (Phase 3 Pitfall 1), keep names short
- Partial UNIQUE INDEX via `DB::statement("CREATE UNIQUE INDEX {short_name}_unique ON {table}({col}) WHERE {col} IS NOT NULL");` — Phase 2/4 D-009 idiom
- Translatable JSONB column: `$table->jsonb('title');` (D-013); HasTranslations sets/reads via getTranslations() in plan 06-03
- FK with cascadeOnDelete: `$table->foreign('parent_id')->references('id')->on('parent_table')->cascadeOnDelete();`
- FK with nullOnDelete: `$table->foreign('related_id')->references('id')->on('related_table')->nullOnDelete();`
- FK with restrictOnDelete: `$table->foreign('frozen_id')->references('id')->on('frozen_table')->restrictOnDelete();`

FK cascade matrix summary (Phase 4 Pattern 1 continuation):

| FK | onDelete | Reason |
|----|----------|--------|
| tournaments.game_id → games | restrict | Don't orphan tournaments when a Game is deleted |
| tournaments.organiser_user_id → users | restrict | Don't orphan tournaments when organiser is deleted |
| tournaments.default_game_match_type_id → game_match_types | nullOnDelete | Tournament can survive removal of its default match type — admin reassigns |
| tournament_participants.tournament_id → tournaments | cascade | Delete tournament → wipe participants |
| tournament_participants.clan_id → clans | restrict | Don't lose participant history when clan is deleted (clan deletes are rare; protect the audit trail) |
| tournament_stages.tournament_id → tournaments | cascade | Delete tournament → wipe stages |
| tournament_brackets.tournament_stage_id → tournament_stages | cascade | Delete stage → wipe its brackets |
| tournament_brackets.participant_a_id/b_id/winner_id → tournament_participants | nullOnDelete | Removing a participant nulls their bracket slot rather than killing the bracket |
| tournament_brackets.match_id → matches | nullOnDelete | A GameMatch deletion (rare; admin only) nulls the bracket's match_id rather than nuking the bracket |
| tournament_brackets.advances_to_bracket_id/loser_advances_to_bracket_id → self | nullOnDelete | Bracket deletion nulls its descendants' advance pointer rather than recursive cascade |
| tournament_standings.tournament_id → tournaments | cascade | Delete tournament → wipe standings |
| tournament_standings.tournament_stage_id → tournament_stages | cascade | Delete stage → wipe its standings |
| tournament_standings.participant_id → tournament_participants | cascade | Delete participant → wipe their standings rows (rebuilds on next recalc anyway) |
