```php

public function down(): void

    DB::statement('ALTER TABLE discord_outbound_messages DROP CONSTRAINT IF EXISTS doutmsg_message_type_chk;');
    DB::statement(
        'ALTER TABLE discord_outbound_messages ADD CONSTRAINT doutmsg_message_type_chk '
```

<!-- FTS trigger function idiom (verbatim from RESEARCH Pattern 3) -->

For each of articles/clans/players, the migration adds a tsvector column, a trigger function, a BEFORE INSERT OR UPDATE trigger, a GIN index, and a backfill UPDATE. Columns indexed:
- articles: title->>'en' + excerpt->>'en' + slug
- clans: name->>'en' + description->>'en' + slug
- players: username + (any public-facing display fields — see Phase 2 Player model). NOTE: respect D-018 by NOT indexing private-only fields (real_name, discord_tag — those are gated by PlayerPrivacyGate at query time in plan 07-08; do NOT index them here because they shouldn't be searchable in the first place).

The `'simple'` text-search config is intentional (RESEARCH A2) — English-only at launch (D-013); no stemming acceptable for v1 editorial corpus.
