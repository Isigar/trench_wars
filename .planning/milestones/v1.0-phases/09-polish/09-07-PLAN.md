## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-07-01 | E (Elevation of Privilege) | Non-moderator triggering bulk-ban | mitigate | Every BulkAction has `->visible(fn () => $user->can('X'))`; defence in depth: BanService::issue could optionally re-check (deferred); ModeratorPermissionGateTest asserts |
| T-09-07-02 | T (Tampering) | Moderator banning admin or other moderator | accept | All moderators trusted; v1 does not introduce ban-tier hierarchy (admin > moderator > user); future enhancement |
| T-09-07-03 | R (Repudiation) | Moderator claims they did not ban a user | mitigate | activity_log row written for every ban (causer=mod, subject=user); append-only per CLAUDE.md s6; never exposed for edit in Filament |
| T-09-07-04 | S (Spoofing) | Service called with wrong causer (e.g., system user) | mitigate | BanService::issue requires User $issuedBy param; controllers always pass auth()->user(); not nullable |
| T-09-07-05 | D (DoS) | Mass ban via BulkAction with 10k users | accept | Filament BulkAction selects current-page records (default 25); bulk over 10k requires explicit "select all" — operationally tolerable |
| T-09-07-06 | I (Information Disclosure) | Dispute body visible to non-participants | mitigate | MatchDisputeResource gated to moderate-disputes permission; never exposed on public surface |
| T-09-07-07 | T (Tampering) | Invalid dispute transition bypassing state machine | mitigate | DisputeService::transition validates state machine; MatchDisputeWorkflowTest asserts all illegal transitions throw |
| T-09-07-08 | D (DoS) | Pitfall 8 — silent BulkAction modal close on validation fail | mitigate | All form fields marked `->required()` + `->minLength` so Filament shows inline errors; Pest test asserts form validation |
