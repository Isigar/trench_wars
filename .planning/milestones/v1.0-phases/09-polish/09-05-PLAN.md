Author `app/Services/LeaderboardService.php` per RESEARCH Pattern 3 (verbatim with refinements):
```
final class LeaderboardService
{
    private const WINDOWS = ['7d' => 7, '30d' => 30, 'all' => null];
    private const TTL_FRESH = 600;
    private const TTL_STALE = 3600;

    public function topPlayers(string $window, ?int $gameId = null, int $limit = 25): Collection { ... }
    public function topClans(string $window, ?int $gameId = null, int $limit = 25): Collection { ... }
    private function computePlayerLeaderboard(string $window, ?int $gameId, int $limit): Collection { ... }
    private function computeClanLeaderboard(string $window, ?int $gameId, int $limit): Collection { ... }
}
```

Implementation:
- `topPlayers()` validates $window in self::WINDOWS keys (throw InvalidArgumentException otherwise). Wraps `Cache::tags(['leaderboards', "lb:players:{$window}"])->flexible($key, [self::TTL_FRESH, self::TTL_STALE], fn () => $this->computePlayerLeaderboard(...))`. Cache key: `lb:players:{$window}:" . ($gameId ?? 'all') . ":{$limit}"`. Pitfall 9 mitigation: wrap the compute callback in try/catch and call `report($e); throw $e;` so failures surface in Horizon.

- `computePlayerLeaderboard()`:
  ```
  MatchPlayerStat::query()
      ->selectRaw('player_id, SUM(kills) AS kills, SUM(deaths) AS deaths, COUNT(*) AS matches_played, (SUM(kills)::float / NULLIF(SUM(deaths), 0)) AS kdr')
      ->when($since, fn ($q) => $q->whereHas('match', fn ($q) => $q->where('scheduled_at', '>=', $since)))
      ->when($gameId, fn ($q) => $q->whereHas('match', fn ($q) => $q->where('game_id', $gameId)))
      ->groupBy('player_id')
      ->orderByRaw('SUM(kills) DESC')
      ->limit($limit)
      ->with(['player.user.privacy', 'player.activeClanMembership.clan'])
      ->get();
  ```
- `computeClanLeaderboard()`: aggregates via `match_player_stats` JOIN `clan_memberships` (active=true at SQL level) JOIN `match_results` (for wins). Refer to RESEARCH Leaderboards SQL block verbatim — translate to Eloquent QueryBuilder using DB::table for readability since this is a JOIN-heavy aggregate (acceptable to use DB::table when Eloquent doesn't simplify). Eager-load via subquery is NOT possible here — return raw rows and let plan 09-06 controller hydrate Clan models for display.
