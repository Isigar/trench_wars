Full schema per RESEARCH Pattern 1 (verbatim — paste columns + indexes + CHECKs):

matches:
  uuid id pk; uuid game_match_type_id (FK restrictOnDelete); jsonb title; jsonb description nullable;
  timestampTz scheduled_at; uuid organiser_user_id (FK restrictOnDelete);
  uuid host_clan_id nullable (FK nullOnDelete); text server_address nullable;
  text status default 'draft' (CHECK IN ('draft','open','locked','played','cancelled'));
  boolean is_public default true; timestamps();
  index('scheduled_at'); index(['status','scheduled_at']); index('is_public');

match_slots:
  uuid id pk; uuid match_id (FK cascadeOnDelete); uuid game_role_id (FK restrictOnDelete);
  integer slot_index; uuid occupant_user_id nullable (FK nullOnDelete);
  timestampTz confirmed_at nullable; integer sort_order default 0; timestamps();
  unique(['match_id','game_role_id','slot_index'], 'match_slots_unique_slot');
  index(['match_id','occupant_user_id']);
  PARTIAL: CREATE UNIQUE INDEX match_slots_one_occupancy_per_user ON match_slots (match_id, occupant_user_id) WHERE occupant_user_id IS NOT NULL;

match_access_rules:
  uuid id pk; uuid match_id (FK cascadeOnDelete); uuid clan_tag_id (FK restrictOnDelete); timestamps();
  unique(['match_id','clan_tag_id'], 'match_access_rules_unique');
  index('match_id');

match_results:
  uuid id pk; uuid match_id unique (FK cascadeOnDelete) [the unique() call makes 1:1];
  uuid winner_clan_id nullable (FK nullOnDelete);
  integer allies_score nullable; integer axis_score nullable; text notes nullable;
  uuid recorded_by_user_id (FK restrictOnDelete); timestampTz recorded_at; timestamps();
  CHECK: ((allies_score IS NULL OR allies_score >= 0) AND (axis_score IS NULL OR axis_score >= 0))

match_mvps:
  uuid id pk; uuid match_result_id (FK cascadeOnDelete); uuid player_id (FK restrictOnDelete);
  text category; integer value nullable; timestamps();
  unique(['match_result_id','category','player_id'], 'match_mvps_unique');
  CHECK category IN ('kills','defense','objective','mvp')

events:
  uuid id pk; string eventable_type; uuid eventable_id;
  timestampTz starts_at; timestampTz ends_at nullable;
  jsonb title; boolean is_public default true; timestamps();
  unique(['eventable_type','eventable_id'], 'events_one_per_owner');
  index(['eventable_type','eventable_id'], 'events_morphable_index');
  index('starts_at'); index(['is_public','starts_at']);

NOTE on `matches.scheduled_at`: per Pitfall 8 + 9, this is `timestampTz` from creation (NOT created as timestamp then ALTER'd). Use `$table->timestampTz('scheduled_at');` directly. Apply the ALTER pattern only to `created_at`/`updated_at` because those are emitted by `$table->timestamps()` as plain timestamp.

NOTE on the `matches` index `is_public` (single-column): Postgres allows it; later observer queries filter by `is_public=true` so a covering index helps the calendar page.
