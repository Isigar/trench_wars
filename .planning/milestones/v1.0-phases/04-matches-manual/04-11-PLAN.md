RESEARCH Pattern 7 Vue skeleton (verbatim Matches/Show.vue):
```vue
<script setup lang="ts">
import PublicLayout from '@/layouts/PublicLayout.vue';
import RoleSlotGroup from '@/components/matches/RoleSlotGroup.vue';
import MatchStatusBadge from '@/components/matches/MatchStatusBadge.vue';
import { useT } from '@/composables/useT';
import { Head } from '@inertiajs/vue3';

const { t } = useT();

type MatchData = App.Data.PublicMatchData;
type RoleGroup = { role: App.Data.GameRoleData; slots: App.Data.PublicMatchOccupantData[] };

const props = defineProps<{
    match: MatchData;
    roleGroups: RoleGroup[];
    signupAllowed: boolean;
    viewerSlotId: string | null;
}>();
</script>

<template>
    <Head :title="match.title?.en ?? t('matches.show.title_fallback')" />
    <PublicLayout>
        <section class="max-w-3xl mx-auto px-4 md:px-6 py-8 flex flex-col gap-8">
            <header class="flex flex-col gap-2">
                <h1 class="font-sans font-semibold text-[28px] leading-[1.2]">
                    {{ match.title?.en ?? t('matches.show.title_fallback') }}
                </h1>
                <MatchStatusBadge :status="match.status" />
                <time class="text-base text-[var(--color-text-muted)]">{{ match.scheduled_at }}</time>
            </header>

            <RoleSlotGroup
                v-for="group in roleGroups"
                :key="group.role.id"
                :group="group"
                :match-id="match.id"
                :signup-allowed="signupAllowed"
                :viewer-slot-id="viewerSlotId"
            />
        </section>
    </PublicLayout>
</template>
```

- Props: `match: PublicMatchData`
- Renders `<a :href="route('matches.show', match.id)">` with hero: scheduled_at date pill (via EventDateBadge), match title (en accessor), game match type label, MatchStatusBadge, slot count summary "{occupied}/{total} signed up" (derived from optional slots prop or computed)

- Renders role header (display_name.en) + grid of SlotOccupantPills + SignupButton if signupAllowed && any slot has displayName === undefined && playerSlug === null (empty slot) AND viewer is not already signed up

SlotOccupantPill.vue (privacy-aware):
- Props: `slot: PublicMatchOccupantData`, `matchId: string`
- Renders:
  - If slot.displayName !== undefined: link to /players/{playerSlug} + display name + ClanTagBadge from clanTag
  - ELSE if slot.clanTag !== undefined: "Anonymous" text + ClanTagBadge (D-008 — clan tag shown)
  - ELSE (empty slot): "Open" placeholder badge

SignupButton.vue:
- Props: `matchId: string`, `gameRoleId: string`, `enabled: boolean` (computed from signupAllowed + role-specific availability)

- signUp(): `router.post(route('matches.signups.store', { match: matchId }), { game_role_id: gameRoleId }, { preserveScroll: true })`
- On error (Inertia's onError callback), surface as a toast or inline error (Phase 2 has no toast system — use Inertia's session.error flash from the controller)
