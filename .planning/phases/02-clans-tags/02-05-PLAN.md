<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Controller to DTO::fromPlayer factory | The boundary where privacy is enforced; bypass = privacy leak |
| DTO to Inertia render | DTO::toArray() outputs the wire JSON the Vue page receives |
| Inertia JSON to Vue | Vue MUST receive an absent key, not a null value (security domain rule) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Information Disclosure | PublicPlayerData construction | mitigate | Use conditional `Optional::create($value)` from spatie/laravel-data (or omit by passing different constructor signatures) so withheld fields are absent in toArray() output — NOT serialised as null. Unit test asserts `array_key_exists('discordTag', $arr) === false` when flag is off. |
| T-02-03-02 | Spoofing | Cross-clan viewer in `show_to=clan` tier | mitigate | PlayerPrivacyGate::viewerInSameClan() queries ClanMembership joining BOTH viewer and player by active membership in the SAME clan_id (intersection, not "any clan in league") |
| T-02-03-03 | Information Disclosure | Cache of DTO across users | accept | DTOs are constructed per-request; no caching. Inertia response per request. |
| T-02-03-04 | Tampering | Direct PlayerPrivacy mutation outside gate | mitigate | Gate methods read PlayerPrivacy via `$player->privacy` relation; trusts Eloquent. Mass-assignment guarded by PlayerPrivacy.$fillable (Phase 1). |
| T-02-03-05 | Repudiation | Privacy-gate decisions | accept | Each request runs through Laravel's logging. Privacy decisions are deterministic functions of state; no need to log per decision. |
</threat_model>

  <action>
    Build the service per Pattern 2 algorithm. The class is stateless — no constructor injection; just methods. Use `?User` typing so guests are first-class (null viewer).

    Algorithm for `passesTier`:
    ```
    if (isOwnProfile($viewer, $player)) return true;
    $tier = $player->privacy?->show_to ?? 'community';
    return match ($tier) {
        'private'   => false,
        'community' => $viewer !== null,
        'clan'      => $viewer !== null && $this->viewerInSameClan($viewer, $player),
        'public'    => true,
        default     => false,
    };
    ```

    Algorithm for `viewerInSameClan`:
    ```
    if ($viewer === null) return false;
    $viewerClanIds = ClanMembership::where('user_id', $viewer->id)->whereNull('left_at')->pluck('clan_id');
    if ($viewerClanIds->isEmpty()) return false;
    $playerClanIds = ClanMembership::where('user_id', $player->user_id)->whereNull('left_at')->pluck('clan_id');
    return $viewerClanIds->intersect($playerClanIds)->isNotEmpty();
    ```
    Use Eloquent collection methods (avoid raw SQL). Eager-loaded relation reuse: if Player passed in has `loadMissing('memberships', 'user.memberships')`, prefer that — but accept the small N+1 in Wave 2; plan 02-07 controller can optimize.

    Algorithm for `allowsSection`:
    ```
    if (isOwnProfile($viewer, $player)) return true;
    $privacy = $player->privacy;
    if ($privacy === null) {
        // Defensive: PlayerPrivacy SHOULD exist (Phase-1 ProvisionFirstLogin creates it), but be defensive
        return false;
    }
    return match ($flag) {
        'show_real_name'     => (bool) $privacy->show_real_name,
        'show_discord_tag'   => (bool) $privacy->show_discord_tag,
        'show_clan_history'  => (bool) $privacy->show_clan_history,
        'show_match_history' => (bool) $privacy->show_match_history,
        'show_stats'         => (bool) $privacy->show_stats,
        default              => throw new \InvalidArgumentException("Unknown privacy flag: {$flag}"),
    };
    ```
