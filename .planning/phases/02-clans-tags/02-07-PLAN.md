<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Unauthenticated visitor to /clans | Pure public — DTO contains only public-safe fields |
| Unauthenticated visitor to /players/{slug} | Privacy gate decides whether the page renders at all |
| Query params `?tag=`, `?q=` | Untrusted input — must not enable SQL injection or IDOR |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Information Disclosure | /clans visible without auth | accept | REQUIRED by REQ-tenancy-multi-clan |
| T-02-04-02 | Information Disclosure | /players/{slug} privacy bypass | mitigate | PlayerPrivacyGate runs FIRST. private → abort(404). community → abort(404) when guest. clan → abort(404) when viewer not in same clan. Note: 404 (not 403) prevents existence disclosure |
| T-02-04-03 | Tampering | ?tag query SQL injection | mitigate | Resolve tag via Eloquent `where('slug', $tag)->first()` — parameter-bound, never raw |
| T-02-04-04 | Tampering | ?q query SQL injection | mitigate | Use Eloquent `where('name', 'ILIKE', '%'.$q.'%')` — parameter-bound |
| T-02-04-05 | Information Disclosure | Member roster on /clans/{slug} | mitigate | Plan 02-07 filters memberships by member.privacy.show_clan_history === true. Withheld members are absent from the DTO collection, not nulled. UI-SPEC `clans.privacy.roster_hidden_partial` notice shown when any member is hidden. |
| T-02-04-06 | Denial of Service | Unbounded /clans list | mitigate | Paginate via Laravel's `->paginate(20)` (UI-SPEC: directory shows cards; no explicit pagination is specified but DDoS protection requires it. Use 20-per-page as the default — matches MemberRow page-size in UI-SPEC). |
| T-02-04-07 | Information Disclosure | Slug enumeration | accept | Slugs are intentionally guessable (`/clans/91st-elite`); no PII; users opt-in via clan create |
</threat_model>
