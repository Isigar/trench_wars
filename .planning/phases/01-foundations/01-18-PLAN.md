**Manual smoke checklist** (per VALIDATION.md "Manual-Only Verifications"):

### A. Filament dual-Tailwind theme renders correctly (Pitfall 1)

This is the **HIGH-risk pitfall** per RESEARCH. The automated tests can't catch CSS regressions.

1. Bring up the stack and rebuild assets:
   ```bash
   make up
   make pnpm ARGS="run build"
   ```
2. In your local `.env` (not committed), fill in real Discord OAuth values from a dev application at https://discord.com/developers/applications. Register `http://localhost:8000/auth/discord/callback` as a redirect URI.
3. Open http://localhost:8000/. Click "Log in with Discord". Complete OAuth. Land logged-in.
4. Run the admin grant from the host:
   ```bash
   make artisan ARGS="trenchwars:make-admin <YOUR_DISCORD_USER_ID>"
   ```
5. Open http://localhost:8000/admin in the same browser session.
6. **Visual checks** (the panel theme must NOT be broken):
   - **Form labels** rendered (not raw HTML defaults)
   - **Input borders** visible
   - **Primary buttons** filled in trench-red (#A4262C)
   - **Table row hover** shows surface-elevated bg
   - **Sidebar nav** lists User, Player, Role, Permission, Audit log
   - **Dark theme** is the default (the panel chrome should match the public site colors — dark olive bg, off-white text)
   - **Light theme toggle** (Filament's built-in) flips to a light variant cleanly
   - No completely-unstyled elements (would indicate Tailwind v3 theme bundle didn't load)
7. **If ANY check fails** — the dual-Tailwind workaround (Pitfall 1) is broken; do NOT mark this checkpoint approved.

### B. Discord OAuth real-world happy path

Pest tests use mocked Socialite. This step exercises the LIVE path against Discord's servers.

1. Log out (POST /auth/logout if you have a logout button OR clear the session cookie).
2. Open http://localhost:8000/, click "Log in with Discord".
3. Verify the Discord OAuth screen authorises with `identify` + `email` scopes.
4. Approve. Verify you land back on `/` with a "Signed in as ..." flash AND a logged-in greeting.
5. Open postgres: `docker compose exec postgres psql -U trenchwars -d trenchwars -c "SELECT discord_id, username, last_login_at FROM users;"` — exactly 1 row.
6. Re-log out, re-log in. Same query: still exactly 1 row, `last_login_at` updated.
7. Verify `players` and `player_privacy` each have exactly 1 row.

### C. /admin/audit shows recent activity

1. As admin, edit a User via /admin/users → change username → save.
2. Open /admin/audit. Verify the "updated" event appears with your causer name.
