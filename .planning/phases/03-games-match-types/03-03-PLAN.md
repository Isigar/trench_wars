<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Controller / API to model fill()/update() | $fillable list is the ONLY arrays-from-request gate; mass-assignment attack surface |
| Filament edit form to model save | Filament respects $fillable; bypasses are by-design (admin) but audited via LogsActivity |
| App code / API / Console to RoleLimit creation | Cross-game invariant has NO DB-CHECK enforcement (would require cross-table Postgres trigger — out of scope per Assumption A6); the model saving() listener is the only programmatic guard |
| App code to LogsActivity | Every CRUD operation logs causer/subject; admin tampering visible in /admin/audit (D-012) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Tampering | Cross-game RoleLimit via API/Console (Pitfall 10) | mitigate | GameMatchTypeRoleLimit::saving() listener throws DomainException when matchType.game_id !== role.game_id; covered by Pest test in this plan |
| T-03-03-02 | Tampering | Mass-assignment on $fillable | mitigate | Strict $fillable list on all 4 models (matches Phase 2 pattern); audit log captures every change |
| T-03-03-03 | Tampering | Composite UNIQUE bypass | mitigate | DB-level UNIQUE INDEX (plan 03-02) is the primary gate; Pest test in this plan asserts QueryException on duplicate insert |
| T-03-03-04 | Tampering | Negative capacity write | mitigate | DB-level CHECK (plan 03-02) is the primary gate; Pest test asserts QueryException on capacity=-1 insert |
| T-03-03-05 | Repudiation | All model mutations | mitigate | LogsActivity trait on every model; logFillable + logOnlyDirty; admin causer auto-captured via Spatie activitylog default (Phase 1 plan 01-14 wiring) |
| T-03-03-06 | Tampering | Cascade delete orphans | mitigate | Migration-layer FK cascade (plan 03-02) handles this; Pest test asserts that deleting a Game cascades to roles + match types + role limits |
</threat_model>
