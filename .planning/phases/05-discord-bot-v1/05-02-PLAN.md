<objective>
Wave 1: ship the two new schema artifacts that Phase 5 depends on — (a) `discord_outbound_messages` durable outbox table that every Discord side-effect (match announces, role syncs) lands in, (b) `clans.discord_announce_channel_id` column resolving RESEARCH Open Question Q1 (channel resolution).

The outbox table is the single durability surface for Discord delivery per RESEARCH Pattern 4. It carries `status pending -> dispatching -> sent | failed` with backoff_until for retry semantics + attempts counter + last_error trace. Bot polls `status=pending` rows, claims them via `status=dispatching + attempts++` in a row-locked transaction (Pattern 4 SQL in plan 05-04), then transitions to `sent` (with sent_message_id) or `failed` (with last_error) on completion.

The DiscordOutboundMessage Eloquent model adds `LogsActivity` (D-012 + CLAUDE.md section 6) so admin can audit every outbound row. `scopeDispatchable` is the canonical query for plan 05-04's poller endpoint (filters out failed rows and backoff-deferred rows in one place).

Purpose: SC-3 (outbound message durability). Without this table the bot has no way to learn about new matches or role syncs the web has dispatched; without backoff_until the failure path can't apply exponential backoff.

Output: 2 migrations + DiscordOutboundMessage model + Clan model amendment + real factory + DiscordOutboundMessageModelTest GREEN.
</objective>

  Schema::create('discord_outbound_messages', function (Blueprint $table) {
      $table->uuid('id')->primary();
      $table->text('channel_id');                    // Discord channel snowflake (text — discord snowflakes overflow JS Number; D-002 precedent)
      $table->string('message_type', 32);            // CHECK constraint enforces allowed values
      $table->string('status', 16)->default('pending'); // pending|dispatching|sent|failed
      $table->jsonb('payload');                      // never null — empty {} for placeholder rows
      $table->unsignedSmallInteger('attempts')->default(0);
      $table->text('last_error')->nullable();
      $table->text('sent_message_id')->nullable();   // Discord message snowflake after successful send
      $table->foreignUuid('causer_user_id')->nullable()->constrained('users')->nullOnDelete(); // human attribution
      $table->timestamp('backoff_until')->nullable();
      $table->timestamps();
      $table->index(['status', 'backoff_until'], 'doutmsg_status_backoff_idx');
      $table->index('message_type', 'doutmsg_type_idx');
  });

  // CHECK constraint via raw DB statement (Schema builder cannot express CHECK)
  DB::statement("ALTER TABLE discord_outbound_messages ADD CONSTRAINT doutmsg_message_type_chk CHECK (message_type IN ('match_announce','role_sync','generic'))");
  DB::statement("ALTER TABLE discord_outbound_messages ADD CONSTRAINT doutmsg_status_chk CHECK (status IN ('pending','dispatching','sent','failed'))");

  namespace App\Models
  use HasFactory, HasUuids, LogsActivity, Notifiable (if needed — likely no)
  protected $fillable: ['channel_id','message_type','status','payload','attempts','last_error','sent_message_id','causer_user_id','backoff_until']
  protected $casts: ['payload' => 'array', 'attempts' => 'integer', 'backoff_until' => 'datetime']
  public function causer(): BelongsTo { return $this->belongsTo(User::class, 'causer_user_id'); }
  public function scopePending(Builder $q): void { $q->where('status', 'pending'); }
  public function scopeDispatchable(Builder $q): void { $q->where('status','pending')->where(function($w){ $w->whereNull('backoff_until')->orWhere('backoff_until','<=', now()); }); }
  public function getActivitylogOptions(): LogOptions { return LogOptions::defaults()->logFillable()->logOnlyDirty()->dontSubmitEmptyLogs(); }
