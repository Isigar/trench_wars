Wave 5: ship the admin-facing surface for Phase 5 — (a) Filament read-only DiscordOutboundMessageResource with a "Retry" action for failed deliveries (RESEARCH Q3 resolution), (b) ClanResource amendment exposing discord_announce_channel_id (RESEARCH Q1 resolution), (c) Artisan commands for issuing + revoking the bot's Sanctum personal access token (RESEARCH Pitfall 3 + Q4 resolution).

The resource is intentionally read-only per CLAUDE.md section 8 + CONTEXT.md ("admin can view + retry failed messages"). Admin never edits payloads or hand-creates rows — both would diverge from the observer-driven invariants. The "Retry" action is a Tables\Actions\Action closure that flips `status=failed -> pending`, zeros `attempts`, clears `last_error` + `backoff_until`. LogsActivity (D-012) captures the admin causer automatically because the model has the trait.

The Artisan commands close Open Question Q4 (bot service user provisioning). The bot service user has a sentinel `discord_id='SYSTEM_BOT'` so it never collides with a real OAuth-provisioned User. Sanctum's `createToken(name, abilities, expires_at)` API returns a `NewAccessToken` with a `plainTextToken` field — that's the value the operator copies into Railway's `WEB_API_TOKEN` env var. Token is shown ONCE (Sanctum stores the SHA-256 hash; the plaintext is unrecoverable after the command exits).

The ClanResource amendment is a single TextInput addition in the existing Profile tab. Filament v3's `TextInput::make('discord_announce_channel_id')->numeric()->maxLength(20)->nullable()` validates as a Discord snowflake.

  namespace App\Filament\Resources
  use Filament\Resources\Resource
  use Filament\Tables\Table
  use Filament\Tables\Columns\TextColumn
  use Filament\Tables\Columns\BadgeColumn
  use Filament\Tables\Filters\SelectFilter
  use Filament\Tables\Actions\Action
  use Filament\Tables\Actions\ViewAction
  use App\Models\DiscordOutboundMessage

  final class DiscordOutboundMessageResource extends Resource {
      protected static ?string $model = DiscordOutboundMessage::class;
      protected static ?int $navigationSort = 22;
      protected static ?string $navigationIcon = 'heroicon-o-paper-airplane';
      protected static ?string $navigationGroup = 'Discord';

      public static function getModelLabel(): string { return __('admin.discord_outbound_message.label'); }
      public static function getPluralModelLabel(): string { return __('admin.discord_outbound_message.plural_label'); }

      public static function form(Form $form): Form {
          // View-only — Filament still requires form() but no fields are editable
          return $form->schema([]);
      }

      public static function table(Table $table): Table {
          return $table
              ->columns([
                  TextColumn::make('created_at')->dateTime()->sortable(),
                  TextColumn::make('message_type')->badge()->color(fn ($state) => match($state) {
                      'match_announce' => 'info',
                      'role_sync' => 'warning',
                      default => 'gray',
                  }),
                  TextColumn::make('status')->badge()->color(fn ($state) => match($state) {
                      'pending' => 'warning',
                      'dispatching' => 'info',
                      'sent' => 'success',
                      'failed' => 'danger',
                  }),
                  TextColumn::make('channel_id')->fontFamily('mono')->limit(20),
                  TextColumn::make('attempts')->numeric()->sortable(),
                  TextColumn::make('last_error')->limit(60)->tooltip(fn ($record) => $record->last_error),
                  TextColumn::make('sent_message_id')->fontFamily('mono')->limit(15)->placeholder('—'),
                  TextColumn::make('causer.username')->placeholder('system'),
              ])
              ->defaultSort('created_at', 'desc')
              ->filters([
                  SelectFilter::make('status')->options([
                      'pending' => __('admin.discord_outbound_message.status.pending'),
                      'dispatching' => __('admin.discord_outbound_message.status.dispatching'),
                      'sent' => __('admin.discord_outbound_message.status.sent'),
                      'failed' => __('admin.discord_outbound_message.status.failed'),
                  ]),
                  SelectFilter::make('message_type')->options([
                      'match_announce' => 'Match announce',
                      'role_sync' => 'Role sync',
                      'generic' => 'Generic',
                  ]),
              ])
              ->actions([
                  ViewAction::make(),
                  Action::make('retry')
                      ->label(__('admin.discord_outbound_message.actions.retry'))
                      ->icon('heroicon-o-arrow-path')
                      ->color('warning')
                      ->visible(fn (DiscordOutboundMessage $record) => $record->status === 'failed')
                      ->action(function (DiscordOutboundMessage $record) {
                          $record->update([
                              'status' => 'pending',
                              'attempts' => 0,
                              'last_error' => null,
                              'backoff_until' => null,
                          ]);
                          activity()
                              ->causedBy(auth()->user())
                              ->performedOn($record)
                              ->event('retry')
                              ->log('admin re-queued failed outbound message');
                          \Filament\Notifications\Notification::make()
                              ->title(__('admin.discord_outbound_message.actions.retry_success'))
                              ->success()
                              ->send();
                      }),
              ])
              ->bulkActions([]); // no destructive bulk actions
      }

      public static function getPages(): array {
          return [
              'index' => Pages\ListDiscordOutboundMessages::route('/'),
              'view' => Pages\ViewDiscordOutboundMessage::route('/{record}'),
          ];
      }
  }

IssueBotTokenCommand shape:

  namespace App\Console\Commands
  use Illuminate\Console\Command
  use App\Models\User

  final class IssueBotTokenCommand extends Command {
      protected $signature = 'trenchwars:bot:issue-token {--name=bot-prod} {--ttl=90}';
      protected $description = 'Issue a Sanctum personal access token for the bot service user (rotation: every 90 days).';

      public function handle(): int {
          $bot = User::firstOrCreate(
              ['discord_id' => 'SYSTEM_BOT'],
              [
                  'username' => 'Trenchwars Bot',
                  'email' => 'bot@trenchwars.local', // unique placeholder
                  // password/email_verified_at via Phase 1 conventions; cite plan 01-09
              ],
          );

          $name = (string) $this->option('name');
          $ttl = (int) $this->option('ttl');

          // Revoke any existing token with the same name (rotation safety)
          $bot->tokens()->where('name', $name)->delete();

          $token = $bot->createToken(
              $name,
              ['bot:read', 'bot:act-as-user', 'bot:write-outbound', 'bot:reconcile'],
              now()->addDays($ttl),
          );

          $this->info('Bot service user: ' . $bot->id);
          $this->info('Token name: ' . $name);
          $this->info('Expires at: ' . $token->accessToken->expires_at->toIso8601String());
          $this->newLine();
          $this->warn('Copy the token below — it is shown ONCE and cannot be recovered.');
          $this->line($token->plainTextToken);
          $this->newLine();
          $this->comment('Paste into Railway bot service WEB_API_TOKEN env var (or apps/web/.env for local dev).');
          $this->comment('Run trenchwars:bot:revoke-token --name=' . $name . ' to revoke.');

          return Command::SUCCESS;
      }
  }

RevokeBotTokenCommand shape:

  protected $signature = 'trenchwars:bot:revoke-token {--name=bot-prod}';
  protected $description = 'Revoke the named Sanctum personal access token for the bot service user.';

  public function handle(): int {
      $bot = User::where('discord_id', 'SYSTEM_BOT')->first();
      if ($bot === null) {
          $this->error('Bot service user not found. Run trenchwars:bot:issue-token first.');
          return Command::FAILURE;
      }
      $deleted = $bot->tokens()->where('name', $this->option('name'))->delete();
      $this->info("Deleted {$deleted} token(s).");
      return Command::SUCCESS;
  }

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-05-07-02 | Spoofing | OAuth user provisions with discord_id='SYSTEM_BOT' colliding with service user | mitigate | Phase 1 plan 01-09 ProvisionFirstLogin uses the Discord-supplied numeric id; 'SYSTEM_BOT' is a sentinel string that no real Discord ID can match (Discord snowflakes are pure digits) |
| T-05-07-03 | Information Disclosure | Sanctum plaintext token displayed in Artisan output, captured in logs | accept | Artisan output flagged with --quiet semantics; operator runs in trusted shell; clip should be cleared (process documentation) |
| T-05-07-04 | Tampering | Operator runs issue-token twice in quick succession leaving 2 valid tokens | mitigate | Command DELETES existing token with same --name before issuing new one (rotation safety) |
| T-05-07-05 | Repudiation | Admin retry action not audited | mitigate | activity()->log('retry') call in the action handler with $record->id + auth()->user() causer |
| T-05-07-06 | Replay | Sanctum token shows in admin Filament resource | mitigate | Sanctum's personal_access_tokens.token column is hashed; even if Admin opens its inspector (no resource ships), the displayed value is the hash, not the plaintext |
| T-05-07-07 | Tampering | Bot service user soft-deleted -> all tokens invalidated | accept | LogsActivity captures; revoke-token + issue-token playbook documented for recovery |
